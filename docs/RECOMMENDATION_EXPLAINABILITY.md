# Recommendation Explainability

## 评分分项

`SongPlanItem.scoreBreakdown` 包含：

- `preferenceAffinity`：导入歌单中的歌手、曲风和情绪偏好。
- `ktvAvailabilityScore`：mock KTV 曲库可唱可用度。
- `vocalFitScore`：歌曲音域与声线稳定音域的重叠。
- `singAlongScore`：多人参与和副歌熟悉度。
- `sceneFitScore`：歌曲标签与朋友局、生日局、团建局、车载 K 歌、情侣局、独自练歌的匹配。
- `varietyScore`：避免曲风过度集中。
- `riskPenalty`：高音、Rap 密度、难度、车载场景注意力压力等扣分。
- `finalScore`：加权求和后扣除风险。

## 硬规则

- 第一首避免高难和高音风险。
- 尽量避免同一歌手连续出现。
- 群体场景保证至少约 30% 合唱友好歌曲。
- 生日局必须补足温暖、祝福或合唱氛围。
- 车载 K 歌降低高难、Rap 密度和注意力负担。
- 锁定歌曲优先保留，移除歌曲不再回到结果。

## 场景策略

- 朋友局：热场、合唱、个人高光、怀旧、收尾。
- 生日局：热身、生日氛围、祝福/合唱、个人高光、收尾。
- 团建局：低门槛、全员参与、气氛推进、熟悉金曲、收尾。
- 车载 K 歌：轻松开唱、熟悉旋律、情绪陪伴、短时高光、合唱收尾。
- 情侣局：轻松开场、甜歌/对唱、情绪段落、个人高光、温暖收尾。
- 独自练歌：开嗓、稳定区练习、技巧挑战、复盘。

## UI 呈现

- 结果页每首歌展示推荐理由、风险提示、替代曲和可展开评分解释。
- 风险提示由真实字段触发，不手写泛泛提醒。
- 替代曲来自相似曲 ID、同歌手或同曲风候选，并在 UI 中作为 chips 展示。
